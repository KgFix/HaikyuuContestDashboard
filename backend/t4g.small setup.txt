Recommended: AWS EC2 t4g.small (Free Tier)
This is a long-running Discord bot that needs to stay online 24/7, making EC2 the most cost-effective option (FREE for 12 months under AWS Free Tier).

Step-by-Step Setup:
1. Launch EC2 Instance
Go to AWS Console → EC2 → Launch Instance
Name: haikyuu-discord-bot
AMI: Ubuntu Server 22.04 LTS (Free tier eligible)
Instance type: t4g.micro (ARM, 1GB RAM - Free Tier: 750 hours/month)
Key pair: Create new or use existing (save .pem file)
Security Group: Default (no inbound ports needed for Discord bot)
Storage: 8GB gp3 (Free tier includes 30GB)

2. Connect to Instance
ssh -i "HaikyuuBotKey.pem" ubuntu@13.50.5.247

3. Install Dependencies
# Update system
sudo apt update && sudo apt upgrade -y

# Install Python 3.11
sudo apt install python3.11 python3.11-venv python3-pip -y

# Install system dependencies for OpenCV and EasyOCR
sudo apt install -y libgl1-mesa-glx libglib2.0-0 tesseract-ocr

4. Setup Bot
# Create directory
mkdir ~/discord-bot
cd ~/discord-bot

# Upload your files (from your local machine):
# Use SCP or WinSCP to transfer files:

On your Windows machine:
scp -i "your-key.pem" -r d:\All\Kirone\Projects\HaikyuuContestDashboardProject\backend\* ubuntu@your-ec2-ip:~/discord-bot/

Back on EC2:
# Create virtual environment
python3.11 -m venv venv
source venv/bin/activate

# Install requirements
pip install -r requirements.txt

# Create .env file
nano .env

Add to .env:
DISCORD_BOT_TOKEN=your_token_here
AWS_ACCESS_KEY_ID=your_key_here
AWS_SECRET_ACCESS_KEY=your_secret_here
AWS_REGION=us-east-1
DYNAMODB_TABLE_NAME=your_table_name

5. Setup as System Service (Auto-restart)
sudo nano /etc/systemd/system/discord-bot.service
Add this content:
[Unit]
Description=Haikyuu Discord Bot
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/discord-bot
Environment="PATH=/home/ubuntu/discord-bot/venv/bin"
ExecStart=/home/ubuntu/discord-bot/venv/bin/python src/discord_bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Enable and start:
sudo systemctl daemon-reload
sudo systemctl enable discord-bot
sudo systemctl start discord-bot

# Check status
sudo systemctl status discord-bot

# View logs
sudo journalctl -u discord-bot -f
